# Warband AI Bridge & Village Engagement Mod

This project integrates Google Gemini AI into Mount & Blade: Warband, allowing for dynamic AI-driven dialogues with NPCs, alongside a custom village engagement system.

##  Features

- **AI-Powered NPC Dialogues**: Seamless integration with Gemini 1.5 Flash for realistic character roleplay.
- **Village Combat System**: Direct "To arms!" engagement from village scenes, jumping straight into the battle menu.
- **Real-time Bridge**: A Python-based file watcher that facilitates communication between the game engine (WSE2) and the Gemini API.

##  Prerequisites & Installation

To run the AI bridge, you need Python 3.x installed on your machine.

### 1. Install Required Libraries
Open your terminal or command prompt and run:

```bash
pip install requests watchdog
```

**What these libraries do:**
- `requests`: Used to send dialogue text to the Google Gemini API and receive the AI's response.
- `watchdog`: Monitors your Warband folder for changes to `To AI Chat.json` so the bridge can react instantly when you talk to an NPC.

### 2. Setup API Key
1. Create a `.env` file in the root directory (never share this file!).
2. Add your Gemini API key:
   ```env
   GEMINI_API_KEY=your_actual_key_here
   ```
3. The bridge uses `os.environ.get("GEMINI_API_KEY")` to securely read this key.

##  How it Works

### The AI Bridge (`ai_gemini.py`)
- **File Watching**: The script watches for a file named `To AI Chat.json` generated by the game.
- **Processing**: When it detects a new message, it sends the NPC's name, kingdom, and your text to Gemini.
- **Output**: The AI's response is written to `From AI Chat.json`, which the game then reads to display the NPC's speech.

### The Village Mod (`module_dialogs.py`)
The "To arms!" dialog option has been upgraded to handle town/village encounters properly. It forces the game to recognize the village as the `$g_encountered_party` and jumps to the attack menu without getting stuck in the world map.

##  Compiling the Mod
1. Place the `module_*.py` files into your Warband Module System folder.
2. Run your local `build_module.bat` to compile the changes into the `.txt` files.
3. Ensure the paths in `ai_gemini.py` (like `WATCH_DIR`) match your actual game installation path.

## ⚠️ Safety Note
- **.gitignore**: The repository is configured to ignore `.env` files to prevent accidental leaking of your API keys.
- **Privacy**: Be aware that dialogue text is sent to Google's servers for processing.
